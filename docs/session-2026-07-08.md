# Session log — 2026-07-08: VapeStation built from scratch

Summary of the Claude Code session that created this project.

## Brief

Build a VST3 graintable synthesizer prototype. Nothing revolutionary — classic,
reliable graintable algorithms. The killer feature: Serum-style modulation,
where every parameter can be modulated by an envelope or LFO. Rough edges
acceptable; no confirmation gates.

## Design decisions

- **JUCE 8.0.14 + CMake** (FetchContent, with a `JUCE_TARBALL` env var to use a
  pre-downloaded tarball). Targets: VST3, Standalone, and a `RenderTest`
  console harness. VST3 auto-installs to `~/.vst3` after build.
- **Engine**: pitch-synchronous granular over wavetable frame-stacks
  ("graintable"): 5 procedural tables (Morph, Sweep, Vox, Bells, Grit), each
  64 single-cycle frames × 2048 samples, generated at startup from additive
  spectra via inverse FFT, band-limited into 4 mip levels (harmonic caps
  64/16/4/1) chosen per grain by playback frequency. Grains are Hann-windowed
  (skewable) table reads, phase-coherent with a voice master phase so the sum
  is clean (COLA) at zero spray. Per-grain randomization: position spray,
  detune, stereo pan. Per voice: SVF filter (LP/BP/HP), ADSR amp envelope.
  10 voices, note stealing, sustain pedal, pitch bend ±2 st.
- **Modulation (the point)**: 8 sources — ENV1 (amp, also routable), ENV2,
  ENV3, LFO1, LFO2 (per-voice, retriggered, 6 shapes), velocity, mod wheel,
  keytrack — routable to all 26 continuous parameters, including envelope
  times and LFO rates themselves. Depth −1..+1 applied in the normalized
  parameter domain, evaluated per voice at control rate (32-sample
  sub-blocks). Base (host) parameter values are never moved.
- **Matrix storage/threading**: routes live as a `MODMATRIX` node inside the
  APVTS state tree (so they persist in plugin state for free). Edits compile
  to an immutable `CompiledMatrix` snapshot swapped in atomically; retired
  snapshots go to a graveyard vector freed at destruction — wait-free for the
  audio thread, no reclamation races.
- **UI**: dark violet theme; the signature element is the modulation halo
  system — drag colored source chips onto any knob to create a route; knobs
  grow colored depth rings plus a live white dot tracking the modulated value
  of the most recent voice; matrix panel edits depths / removes routes;
  right-click a knob for per-route removal; graintable display doubles as a
  Position control; on-screen MIDI keyboard. A default LFO1→Position route
  ships so modulation is audible immediately.

## Verification (all passing)

1. **RenderTest** (offline MIDI render through the real processor), 12/12:
   silence without notes; note produces audio; finite output; release decays
   to true silence; bitwise-deterministic renders (seeded RNGs); an
   LFO→position route changes output (relative L2 diff ≈ 1.09); param and
   matrix state round-trip; demo wav + editor PNG snapshot written.
2. **pluginval** strictness 10: SUCCESS (parameter fuzzing, background-thread
   state, parameter thread safety, editor automation).
3. **Live end-to-end on the desktop**: launched the Standalone, clicked the
   on-screen keyboard via xdotool while recording the audio-interface monitor
   (real audio captured: RMS 0.019); dragged the LFO2 chip onto Cutoff with
   synthetic mouse — route row + green mod ring appeared; held a note and saw
   the position marker animate with the LFO; removed the route via the x
   button (exercises the deferred-delete path) — clean.

## Bugs caught and fixed during the session

- Table synthesis initially used cosine phases (real coefficients), so the
  "saw" frame rendered as a cusp/"bathtub" shape — caught by looking at the
  UI snapshot. Fixed by rotating coefficients by −i (sine-phase synthesis),
  which also lowers crest factor.
- The matrix row remove button used a `×` glyph the default font couldn't
  render; replaced with plain `x`.
- (Test-harness only) a first synthetic remove-click missed due to
  double-counting the JUCE standalone's 28px title bar — app was fine.

## Known rough edges

No LFO tempo sync; no unison; mip switching can step timbre between notes at
extreme transpose; matrix depths aren't host-automatable parameters; fixed
1000×735 UI; one benign deprecation warning in RenderTest
(`createWriterFor`).

## Portability notes

The Linux `.so` won't run on macOS. On a Mac the project should build as-is;
use `-DCMAKE_OSX_ARCHITECTURES="arm64;x86_64"` for a universal binary
(Apple Silicon + Intel in one build). Adding AU is a one-word change to
`FORMATS` in CMakeLists. Unsigned builds are fine locally; signing/
notarization only matters for distribution.
